# Backend Java Standards - TalentVillage.ai

## Runtime & Framework

- **JDK**: Amazon Corretto 17+
- **Framework**: Spring Boot 3.x
- **Build Tool**: Gradle (Kotlin DSL)

## Coding Standards

- Follow Google Java Style Guide.
- Use Records for DTOs and immutable data structures.
- Favor `Optional` over null returns.

## Required Skills

### Core Java Expertise
- **Language Proficiency**: Deep understanding of Java 17+ features including Records, Sealed Classes, Pattern Matching, and Text Blocks
- **Concurrency**: Expert knowledge of multithreading, CompletableFuture, virtual threads (Project Loom), and reactive programming
- **Memory Management**: Understanding of JVM internals, garbage collection tuning, and performance optimization
- **Functional Programming**: Proficiency with Streams API, lambda expressions, and functional interfaces

### Spring Ecosystem
- **Spring Boot**: Advanced knowledge of auto-configuration, actuators, profiles, and production-ready features
- **Spring Core**: Deep understanding of dependency injection, AOP, and Spring lifecycle
- **Spring Data**: Experience with JPA/Hibernate, Spring Data repositories, and query optimization
- **Spring Security**: OAuth2, JWT, role-based access control, and security best practices
- **Spring Cloud**: Microservices patterns, service discovery, API gateways, and distributed tracing

### Database & Persistence
- **Relational Databases**: PostgreSQL, MySQL optimization, indexing strategies, and transaction management
- **ORM**: Hibernate/JPA entity relationships, lazy loading, caching (L1/L2), and N+1 query prevention
- **NoSQL**: MongoDB, Redis for caching, and understanding of when to use document vs relational databases
- **Migrations**: Flyway or Liquibase for database version control

### API Design & Integration
- **RESTful APIs**: Richardson Maturity Model, HATEOAS, versioning strategies
- **API Documentation**: OpenAPI/Swagger specification and documentation generation
- **GraphQL**: Schema design, resolvers, and query optimization (optional but valuable)
- **Message Queues**: RabbitMQ, Kafka for event-driven architectures

### Testing & Quality
- **Unit Testing**: JUnit 5, Mockito, test-driven development practices
- **Integration Testing**: TestContainers, Spring Boot Test, API integration testing
- **Code Quality**: SonarQube, static analysis tools, and maintaining >80% test coverage
- **Performance Testing**: JMeter, Gatling for load testing

### Build & Development Tools
- **Gradle**: Kotlin DSL, dependency management, custom tasks, and multi-module projects
- **Maven**: Alternative build tool knowledge
- **IDEs**: IntelliJ IDEA proficiency, debugging techniques, and profiler usage
- **Version Control**: Git workflows, branching strategies, code review practices

### Architecture & Design
- **Design Patterns**: SOLID principles, GoF patterns, and architectural patterns (MVC, CQRS, Event Sourcing)
- **Microservices**: Service decomposition, domain-driven design, and distributed system challenges
- **Clean Architecture**: Hexagonal architecture, dependency inversion, and bounded contexts
- **API Security**: Rate limiting, CORS, CSRF protection, and input validation

### DevOps & Production
- **Containerization**: Docker, container optimization, and multi-stage builds
- **CI/CD**: Jenkins, GitHub Actions, automated testing and deployment pipelines
- **Monitoring**: Prometheus, Grafana, application metrics, and logging (SLF4J, Logback)
- **Cloud Platforms**: AWS (ECS, Lambda, RDS), Azure, or GCP deployment experience

### Soft Skills
- **Problem Solving**: Debugging complex issues, performance bottlenecks, and system design challenges
- **Communication**: Technical documentation, API documentation, and collaboration with frontend teams
- **Code Review**: Providing constructive feedback and maintaining code quality standards
- **Mentorship**: Guiding junior developers and sharing knowledge
